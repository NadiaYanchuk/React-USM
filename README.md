## Лабораторная работа №1

**Цели:**
1. Научиться создавать базовые компоненты,
2. Использовать хук состояния useState,
3. Использовать хук эффектов useEffect,
4. Написать простой CRUD (create / read / update / edit) над сущностью данных.

**Задачи:**
1. Должен отображаться на экране список сущностей (useEffect + fetch). Хорошо, если будет реализована пагинация данных, или ленивая загрузка.
2. Функционал просмотра сущности. При нажатии на элемент списка открывается модальное окно (div) с дополнительной информацией о сущности.
3. Редактирование. В окне просмотра можно редактировать свойства сущности и сохранить их, при этом в списке она будет обновлена.
4. Создание. При нажатии на кнопку _«Создать»_ или _«Добавить»_ будет появляться форма полей, при заполнении всех свойств, сущность будет создана и добавлена в список. (Форма редактирования и создания, по сути, одна и таже, только с возможностью принимать сущность через props или создавать пустую)
5. Удаление. Удаляет сущность из списка сущностей.


## Лабораторная работа №2

**Цели:**
1. Научиться создавать поиск по данным (по названию/полям).
2. Реализовать фильтры (сортировка, диапазоны, чекбоксы и т.п.)
3. Освоить оптимизацию рендера: мемоизация результатов фильтрации.

**Задачи:**
1. Поле ввода _«Поиск …»_. Фильтрация по названию и / или по описанию. Поиск должен быть моментальным, при вводе, список должен обновляться и содержать только те элементы, которые попадают под критерии поиска.
2. Фильтры. Ваша сущность может содержать чётко определённые свойства (тип товара или пользователь написавший пост), эти свойства могут быть одинаковы для нескольких, по ним можно сделать сортировку. Диапазон данных можно использовать для цены, рейтинга либо даты.  Реализуйте как сортировку по чётко определённым значениям, так и по диапазонам данных.
3. Состояния должны быть добавлены дополнительно для информирования о статусе фильтрации и управлении:
- Показать лоадер при загрузке.
- Показать _“Ничего не найдено”_ при пустом результате.
- Кнопка _“Сбросить фильтры”_.
- Кол-во найденных элементов _“Найдено: X”_.


## Лабораторная работа №3

**Цели:**
1. Научится использовать ContextAPI для хранения сущностей и передачи сущностей
2. Реализовать маршрутизацию с помощью react-router-dom
3. Научиться передавать данные между страницами без перезагрузки и без props drilling.

**Задачи:**
1. Создать контекст сущностей (к примеру ProductsContext), который будет иметь следующие методы:
- `fetchEntities()` — загрузка данных с API (выполняется один раз);
- `getById(id)` — получить сущность по идентификатору;
- `addEntity(entity)` — добавить новую сущность;
- `updateEntity(entity)` — обновить данные существующей;
- `deleteEntity(id)` — удалить из списка.
Контекст должен быть доступен во всех страницах приложения (через провайдер).
2. Маршрутизация. Использовать react-router-dom со следующими путями: 
      `/`
      `/entity/:id`
      `/create`
      `/edit/:id`
3. Работа с данными через контекст. Все операции (просмотр, редактирование, добавление, удаление) выполняются через методы контекста, без локальных списков в компонентах.
